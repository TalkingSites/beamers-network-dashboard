Before writing any code, talk through the feature or phase with the user.

1. Describe what you're planning to build in plain language — what pages, what interactions, what data flows
2. Call out any design or UX decisions that need the user's input before you can start
3. Flag anything that has architectural implications (new DB fields, new routes, external APIs, etc.)
4. Ask the user if anything is missing, wrong, or should be done differently
5. Wait for the user to confirm before writing a single line of code

Keep it conversational, not a formal spec. The goal is alignment, not documentation.
